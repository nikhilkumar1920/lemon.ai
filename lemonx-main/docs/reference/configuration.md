# Configuration

Reference for all configuration options in lemon.test.

## Environment Variables

### Required

| Variable | Description | Example |
|---|---|---|
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare account ID for Workers AI | `abc123def456` |
| `CLOUDFLARE_API_KEY` | Cloudflare API token | `your-api-key` |
| `REDIS_HOST` | Redis server hostname | `redis` |
| `REDIS_PORT` | Redis server port | `6379` |

### Optional (for PR creation)

| Variable | Default | Description |
|---|---|---|
| `TARGET_REPO` | `process.cwd()` | Path to the target repository |
| `LEMON_WORKSPACE` | — | Working directory (set automatically by webhook mode) |
| `WEBHOOK_PORT` | `3456` | Port for the webhook server |
| `WEBHOOK_SECRET` | — | HMAC-SHA256 secret for webhook verification |
| `CIRCLECI_RUNNER_NAME` | `lemon-runner` | Name for the CircleCI machine runner |
| `CIRCLECI_RUNNER_API_AUTH_TOKEN` | — | Authentication token for the CircleCI runner |

### Required for PR Creation

| Variable | Description | Example |
|---|---|---|
| `GITHUB_TOKEN` | GitHub personal access token with `repo` scope | `ghp_xxxx` |
| `GITHUB_REPOSITORY` | Owner/repo format | `owner/my-repo` |
| `GITHUB_REF` | Git ref for the base branch | `refs/heads/main` |
| `GITHUB_SHA` | Full commit SHA | `abc123def456789...` |

## Constants

### MAX_ITERATIONS

**Value**: `5`

**Description**: Maximum number of fix loop iterations per test type. After this many iterations, the loop stops even if tests are still failing.

**Location**: `src/index.ts` and `src/webhook-server.ts`

---

## File Discovery Configuration

### Unit Test Discovery

**Filter**: All `.ts`/`.js` files
**Exclusions**: `node_modules`, `__tests__`, `.d.ts`, `seeds/`, `migrations/`, `public/`
**Limit**: 5 files

### Integration Test Discovery

**Filter**: Files containing `routes`, `api`, `service`, `controller`, `middleware`, or `handler`
**Exclusions**: `node_modules`, `__tests__`, `.d.ts`
**Limit**: 5 files

### E2E Test Discovery

**Filter**: Files containing `app`, `server`, `index`, `routes`, or `auth`
**Exclusions**: `node_modules`, `__tests__`, `.d.ts`
**Limit**: 3 files

## CircleCI Configuration

### Example Repo Config (generated by lemonx)

```yaml
version: 2.1

workflows:
  ai-test-and-fix:
    jobs:
      - ai-test-loop:
          filters:
            branches:
              ignore: main

jobs:
  ai-test-loop:
    machine: true
    resource_class: <namespace>/<resource-class>
    working_directory: ~/project
    steps:
      - checkout
      - run:
          name: Run lemon.test AI test-fix loop
          command: |
            RESPONSE=$(npx tsx /home/circleci/lemon/src/index.ts 2>&1)
            echo "$RESPONSE"
            if echo "$RESPONSE" | grep -q "⚠️.*Max iterations reached"; then
              exit 1
            fi
            if echo "$RESPONSE" | grep -q "❌"; then
              exit 1
            fi
```

### Required Replacements

After running `npx lemonx init`, you must replace `<namespace>/<resource-class>` with your actual CircleCI runner resource class (e.g., `my-org/lemon-runner`).

## Docker Configuration

### docker-compose.runner.yml (Recommended)

Starts:
- **Redis** (redis:7-alpine) with health check
- **Runner** (CircleCI machine runner with lemon.test)

### docker-compose.yml (Webhook Mode)

Starts:
- **Redis** (redis:7-alpine) with health check
- **App** (direct execution mode)
- **Webhook** (Express server on port 3456)
